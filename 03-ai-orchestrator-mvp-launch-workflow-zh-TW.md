用多工具、多模型在有限時間內交出可 Demo 的作品

## 0. 系列地圖：本篇在整體中的位置

本篇是系列文的「實作 SOP」：

- 第 1 篇：定義 R0 / SSOT / Core Kernel + Modules。  
- 第 2 篇：用 L1–L10 描述 AI 使用成熟度，幫助溝通。  
- 第 3 篇（本篇）：說明從題目選定 → MVP → Demo → 部署 → 上線溝通 → 迭代的完整流程。  
- 第 4 篇：將這套流程實際用在 Kaggle 教育案例 GapLens。

可以把本篇想像成：

> **「如何在 4–8 週內，用多個 AI 工具完成一個可 Demo 的產品」的標準作業流程。**

---

## 1. 使用情境與角色

### 1.1 適用對象

這條工作流主要適用於：

- 想把 side project 做成可以在履歷／面試上展示的 **AI 作品**。  
- 需要在部門專案中，帶頭做一個「AI＋自動化」改善案。  
- 準備往 **AI Orchestrator / AI Workflow Architect / AI PM** 角色發展的人。

### 1.2 典型題目範例

- 訂閱管理工具、數位資產庫、個人財務儀表板。  
- 城市情報地圖（噪音、嫌惡設施、寵物友善地點…）。  
- 教育工具：像 GapLens 這種「學習落差掃描器」。  
- 部門績效與獎酬制度分析工具。  

---

## 2. 工具與分工（不是亂用，而是分層用）

### 2.1 規劃與結構層

- **ChatGPT**  
  - 需求拆解、流程設計、資料結構（JSON / schema）草稿、PRD 與技術說明、Prompt 設計。
- **Claude**  
  - 閱讀／重構程式碼、協助改寫較大段實作、找 bug。
- **Perplexity**  
  - 市場／競品／行為研究，支撐設計與商業假設。
- **NotebookLM / 其他知識庫工具**  
  - 收納專案文件、決策與程式片段。

### 2.2 UI / 視覺層

- **Google AI Studio + Gemini 3 Pro**  
  - 快速生成 App / Web 假畫面與操作流程。  
  - 對黑客松或概念驗證特別有用。

### 2.3 影片與聲音層

- **Sora / Kling / 類似影片模型**：產生情境或操作影片。  
- **Suno**：背景音樂。  
- **TTS 工具**（例如 TTSMaker）：將腳本轉成多語配音。  
- **Canva 等剪輯工具**：剪輯 Demo 影片、加字幕與說明卡。

### 2.4 在地化與文化層

- **在地語言模型（例如豆包）**：將文案調整為符合當地語境的版本，用於多市場敘事對照。

## 3. 七階段 MVP → 上線工作流

以下是我實際使用的七個階段。  
每一階段會說明：

- 目標  
- R0 要做什麼  
- 會用到哪些 AI 工具  
- 建議產出物

---

### 第 0 階段：題目與方向確認

**目標**  
選定這一輪要做的題目，鎖定目標族群與時間箱（例如 4–8 週）。

**R0 的任務**

- 在 SSOT 中建立「實驗卡」，包含：  
  - 題目名稱與類別（例如 Education / Finance / City Intel）；  
  - 目標族群（老師、家長、個人理財者…）；  
  - 時間範圍與最小成功條件（Demo 影片 + 一個可分享連結）。

**AI 的使用方式**

- ChatGPT：蒐集並整理可能的題目，評估哪一題較符合個人優勢。  
- Perplexity：查競品與現有解法，避免做出完全一樣的東西。

---

### 第 1 階段：Problem / Market Fit 初步檢查

**目標**  
確認題目值得投入做 MVP，而不是只是一個「好像很酷」的想法。

**R0 的任務**

- 將核心假設與成功／失敗條件寫入 SSOT：  
  - 例如：「如果 4–6 週內有 N 位目標使用者願意看 Demo 並給具體回饋，視為通過」。

**AI 的使用方式**

- Perplexity：整理目標族群目前的痛點與常見解法。  
- ChatGPT：將資料整理成一頁「市場快照」＋「我的切入角度」。

**產出建議**

- 一份 Markdown / Notion 頁面，可直接引用到 README 或案例說明。

---

### 第 2 階段：MVP 與體驗流程設計

**目標**  
用文字（而不是 Figma）把 v0.1 的產品說清楚，只專注在一個最重要的工作情境。

**R0 的任務**

- 在 SSOT 中：  
  - 指定或新增模組（例如 `M04_Education_GapLens`）。  
  - 定義這一輪要用到的核心資料欄位。  
  - 定義模組與 Core 的介面與授權。

**AI 的使用方式**

- ChatGPT：  
  - 產生 MVP 功能清單（只保留 1–2 個關鍵功能）。  
  - 寫出 step-by-step 使用者流程（User Story + User Flow）。  
- ChatGPT（不同 Persona 設定）：  
  - 模擬老師、學生、家長等不同角色走一次流程，找出可能卡關點。

**產出建議**

- 一份「MVP 使用劇本 + 流程圖」，讓設計、工程與評審都能秒懂產品在做什麼。

---

### 第 3 階段：Prototype（假 UI）與 Demo 故事線

**目標**  
在寫程式前，先讓所有人看得懂產品長什麼樣、怎麼操作。

**R0 的任務**

- 確認畫面流程與資料流一致；  
- 確保畫面上不混入內部註解或 JSON。

**AI 的使用方式**

- Google AI Studio + Gemini 3 Pro：  
  - 生成互動式 App 假畫面，包含輸入區、結果區、摘要區。  
- ChatGPT：  
  - 撰寫 Demo 旁白腳本與分鏡（搭配第 4 篇的 GapLens 案例）。

**產出建議**

- 一個可以操作的 AI Studio App Prototype；  
- 一份對應的 Demo 旁白腳本。

---

### 第 4 階段：Demo 影片製作

**目標**  
產出一支 60–120 秒的 Demo 影片，可以放在 Kaggle、GitHub、面試或社群平台。

**R0 的任務**

- 確保影片不誇大功能，Demo 內容在合理技術範圍內。  
- 決定影片的主軸：是教育、是技術展示、還是 PM 敘事。

**AI 的使用方式**

- TTS 工具：將英文或中文旁白腳本轉成配音。  
- Suno：產生不干擾理解的背景音樂。  
- Canva 等剪輯工具：  
  - 把螢幕錄影、配音、字幕與註解卡片剪在一起。

---

### 第 5 階段：開發與部署

**目標**  
實作出真正可用的 MVP，並部署到可分享的網址（如 Vercel、Cloudflare Pages）。

**R0 的任務**

- 更新 SSOT：  
  - schema（資料表結構）  
  - API 介面  
  - `modules_map` 與 `integration_registry`  
- 審查 AI 產出的程式碼是否符合：  
  - 模組邊界、  
  - 安全與維護性要求。

**AI 的使用方式**

- ChatGPT：  
  - 協助設計資料表、撰寫初版程式碼與測試案例。  
- Claude：  
  - 協助閱讀、重構與除錯。  

---

### 第 6 階段：上線說明與對外溝通

**目標**  
讓目標使用者、主管或評審理解這個工具在做什麼，以及為何重要。

**R0 的任務**

- 決定這一輪上線的主要目的：  
  - 收集回饋？  
  - 作為作品集？  
  - 邀請合作開發？

**AI 的使用方式**

- ChatGPT：  
  - 撰寫 README、Landing Page、社群貼文與簡短標語。  
- Perplexity：  
  - 提供相關領域的關鍵字與常見搜尋語，優化 GitHub SEO。

---

### 第 7 階段：數據與回饋 → 下一步決策

**目標**  
根據使用數據與質性回饋，決定下一步是：持續、轉向還是暫停。

**R0 的任務**

- 在 SSOT 中記錄：  
  - 實際達成的成果（影片、連結、使用者回饋）。  
  - 與原本成功／失敗條件的差距。  
- 規劃下一輪實驗（可能是同一題目更深一層，或是換題目）。

**AI 的使用方式**

- ChatGPT：  
  - 協助整理問卷與訪談紀錄，歸納主要 insight。  
- 其他分析工具：  
  - 依需求加入簡單的事件追蹤與儀表板。
