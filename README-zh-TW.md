## 1. 概要說明

我以 **AI Orchestrator / AI Workflow Architect** 的方式工作，  
專注設計「人員、資料、系統與多個 LLM」如何在 **SSOT（Single Source of Truth）** 之下協同運作。

背景結合：

- 企業顧問經驗（財務規劃／系統導入），  
- 模組化系統設計（Core Kernel + Modules），  
- 行為與認知心理對產品／工作流設計的應用。

此 repo 彙整：

- 我的 **R0 / AI Orchestrator 方法論**，以及  
- 實際專案中的應用案例。  

用來作為應徵 **AI 相關 PM／策略／工作流架構師** 職務時的作品集。

## 3. 專案案例（Case Studies）

### 3.1 Remote Life OS (P70) – 遠距生活營運系統

- **角色：** Product Owner / R0 Architect / AI Orchestrator  
- **範圍：**
  - 為遠距工作者與數位游牧設計的生活營運工具，  
  - 城市情報地圖、  
  - 訂閱與副業整合。  
- **重點：**
  - 以 SSOT 設計 Core Kernel + Modules 架構，  
  - 使用 L1–L10 模型協調多個 LLM 做架構、實作與 QA，  
  - 實作出地圖 Kernel、訂閱視圖與副業實驗的 MVP。

### 3.2 Side Hustle Value Dashboard – 副業時間與價值儀表板

- **角色：** Product Strategist / AI Orchestrator  
- **範圍：**
  - 副業的時間、能量、財務與學習價值追蹤，  
  - 以 4–8 週實驗周期為核心。  
- **重點：**
  - 定義副業 Slot、Session、Value、Experiment 的結構化 schema，  
  - 使用 AI 一起設計 KPI、儀表板與報表敘事，  
  - 實際套用在自己的多個副業題目上。

### 3.3 City Intel Map Kernels – 城市情報地圖核心

- **角色：** System Designer / R0 Architect / AI Orchestrator  
- **範圍：**
  - 以地圖為核心的決策工具（嫌惡雷達、寵物友善地圖、土地／開發雷達等），  
  - 多個主題共用同一套城市／地點資料結構。  
- **重點：**
  - 設計可重複使用的 City Intel Kernel，讓不同模組可以插上去，  
  - 以標籤與屬性系統整合 UGC 與主題視角，保持核心穩定，  
  - 使用 AI 協助 schema 設計、UX 文案與情境設計。
